# CRON Job
###########


****

* * * * * command_to_run
│ │ │ │ │
│ │ │ │ └── Day of Week (0–7) → Sunday = 0 or 7
│ │ │ └──── Month (1–12)
│ │ └────── Day of Month (1–31)
│ └──────── Hour (0–23)
└────────── Minute (0–59)

*/5 * * * * -> per 5 min

10 2 * * * -> every day 2:10 AM

30 2 1 * * -> every months 1st day 2:30 AM

30 2 1 1 * -> 1st january 2:30 AM

30 2 1 1 1 -> when January 1st is a Monday at 2:30 AM.


crontab -e == edit 
crontab -l == list

# Operator in Bash
#####



| Operator | Meaning     | Example     | Result |
| -------- | ----------- | ----------- | ------ |
| `+`      | Addition    | `$((5+3))`  | 8      |
| `-`      | Subtraction | `$((10-4))` | 6      |
| `*`      | Multiply    | `$((3*4))`  | 12     |
| `/`      | Divide      | `$((8/2))`  | 4      |
| `%`      | Modulus     | `$((10%3))` | 1      |



| Operator | Meaning          | Example           |
| -------- | ---------------- | ----------------- |
| `-eq`    | Equal            | `[[ $a -eq $b ]]` |
| `-ne`    | Not equal        | `[[ $a -ne $b ]]` |
| `-gt`    | Greater than     | `[[ $a -gt $b ]]` |
| `-lt`    | Less than        | `[[ $a -lt $b ]]` |
| `-ge`    | Greater or equal | `[[ $a -ge $b ]]` |
| `-le`    | Less or equal    | `[[ $a -le $b ]]` |


| Operator | Checks              |
| -------- | ------------------- |
| `-e`     | File exists         |
| `-f`     | Regular file exists |
| `-d`     | Directory exists    |
| `-r`     | Readable            |
| `-w`     | Writable            |
| `-x`     | Executable          |
| `-s`     | File is NOT empty   |


| Situation           | Use       |
| ------------------- | --------- |
| Math                | `(( ))` ✅ |
| Strings             | `[[ ]]` ✅ |
| Files               | `[[ ]]` ✅ |
| Multiple conditions | `[[ ]]` ✅ |

# Chmod permission Chart
#####

| Number | Permission | Meaning                |
| ------ | ---------- | ---------------------- |
| `7`    | `rwx`      | Read + Write + Execute |
| `6`    | `rw-`      | Read + Write           |
| `5`    | `r-x`      | Read + Execute         |
| `4`    | `r--`      | Read only              |
| `3`    | `-wx`      | Write + Execute        |
| `2`    | `-w-`      | Write only             |
| `1`    | `--x`      | Execute only           |
| `0`    | `---`      | No permission          |

| Command     | Meaning                                       |
| ----------- | --------------------------------------------- |
| `chmod 777` | Everyone full access ⚠️ Dangerous             |
| `chmod 755` | Owner full, others read+execute ✅ Very common |
| `chmod 700` | Only owner access 🔐 Secure                   |
| `chmod 644` | Owner write, others read ✅ Used for files     |
| `chmod 600` | Owner read/write only 🔐 Password files       |


u = user
g = group
o = others
a = all

 When something breaks at 2 AM, Googling isn’t fast enough.

These Linux commands help engineers diagnose & fix issues in minutes 👇

🚨 systemctl --failed — Instantly find broken services

📄 journalctl -p 3 -xb — Show only critical system errors

📉 top → press 1 — Per-CPU usage (find overload fast)

🧠 free -m && swapon --show — Memory & swap pressure check

💽 df -i — Check inode exhaustion (hidden disk full issue!)

📂 du -ah / | sort -rh | head — What’s really eating disk

🌐 ss -plant — See blocked or stuck connections

🔍 lsof | wc -l — Detect file descriptor leaks

⚠️ uptime — Spot system overload in 2 seconds

🔁 systemctl restart service — Quick recovery move
